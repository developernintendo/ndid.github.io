Step 1 Get Familiar with Wii U

Wii U Overview

The Wii U Overview document covers the various features available to application developers. Topics covered include:

  • Hardware
  • Software
  • System features
  • Communication features
  • Mii
  • e-Commerce
  • Patches

Reading through the Overview materials is a good way to get familiar with the system and the various features and services available to your application. No matter if you are starting a brand new project from scratch or are porting an existing application, the information in the Overview will help with the planning and design process.

  • Wii U Overview
Wii U Guidelines

All applications which are to be released on a Nintendo platform must pass our Lotcheck certification process. The Lotcheck group tests each and every application using the same set of rules, which are called Guidelines. The Guidelines are designed to help application developers provide a safe, friendly, and consistent experience for all users of Wii U software. This is done by defining a minimum set of functional and operational requirements, each of which is paired with a step-by-step testing process.

The Lotcheck process is not QA; instead the Lotcheck group tests each application for specific operational requirements. Guidelines cover a range of topics, such as:

  • General application behavior and design
  • Communications features
  • E-Commerce
  • Safety
  • And more…

We encourage everyone to read through and understand the Guidelines and requirements early in the development process so that application features can be planned accordingly.

  • Wii U Guidelines

Step 2 Select a Wii U Development Option

Nintendo Web Framework

Nintendo Web Framework (NWF) enables developers to create games and web services for Wii U using HTML5, Javascript, CSS and other proven web technologies without expertise in lower-level programming languages such as C. NWF includes support for the Wii U GamePad controller, Wii Remote controllers, and JavaScript extensions.

NWF includes an integrated interface (dashboard) for executing, testing, and debugging your game or application. The dashboard lets you analyze source code while verifying the results in real time.

  • Nintendo Web Framework Documentation
Cafe SDK

Cafe is the code name for Wii U. Cafe SDK, the native SDK for Wii U development, is used for developing applications directly in C or C++.  By programming close to the hardware applications can achieve the fastest possible performance.  If you are porting an existing C++ codebase to Wii U, be sure to read up on the threading system and compiler to determine capability with your existing code.  Applications built using the native SDK can potentially utilize all of the Wii U system features available in the SDK.  Application development middleware packages are available to supplement the Cafe SDK, including NintendoWare, which provides integrated tools and libraries that can be used for graphics and audio development.  The SDK and related Nintendo provided add-on libraries are provided free to Nintendo developers.

See the Wii U Online Documents for more information on the SDK and available packages.

  • Wii U Online Documents

Step 3 Examine Middleware Options

Middleware

Middleware refers to tools that can simplify and aid your game development, providing services as diverse as game engines, compression tools, video codecs, sound players, and more. In addition to a selection of 3rd Party Middleware available for our systems, Nintendo also has a selection of Middleware that we have licensed for free or reduced cost to our developers. Some Middleware are offered in the North America/Europe regions, as well as the Japanese region, but some are only available in specific regions. Please follow the quick links to see our selection of Middleware and find the ones that suit your development needs.

Step 5 Verify Host PC Requirements

Requirements when using CAT-DEV hardware

Development using CAT-DEV hardware has the following hardware and software requirements.  Please confirm that your host PC meets the requirements before getting started:

  • Minimum PC Requirements
    • Windows 7 Professional 64-bit
    • A User Account with Administrator Privileges
    • A 2Ghz Core 2 Duo processor or better
    • At least 1GB of RAM
    • At least 40 GB Hard Disk space, NTFS-formatted

Step 7 Publisher Website Account Access and Project Sheet

How do I get started?

Because you are here, you are already licensed as a developer for a Nintendo platform.

To become a publisher, so you can release a game on the Nintendo eShop, you must know and understand the terms of your publisher agreement with Nintendo. We want to make sure there are no surprises by having you sign an agreement before you place your content on Nintendo eShop.

Depending on the region you wish to release to, you will have to work with either Nintendo of America, Nintendo of Europe or Nintendo Co. Ltd.  Guides to help you get started are listed in the quick links.

  • NOA Publishing Guide
  • NOE Publishing Guide
Common Questions

Is there concept approval?

  • There is no concept approval that developers have to go through to self-publish on Nintendo eShop.
  • We do have programing and content guidelines, and at the end of the development process, games go through our Lotcheck certification process where we confirm that the game is consistent with all of our guidelines.
  • If you want to release free content, please share your plans with Nintendo.

Can I self-publish my game?

  • An intermediary publisher is not required between a developer and Nintendo.
  • Nintendo acts as an agent on your behalf (Commission Agent Model).

Can I set the price, release date and time?

  • Pricing is flexible and can be changed (increased or decreased) by you
  • Temporary changes (sales) can be scheduled by you
  • Release date and release time can be scheduled by you

What percentage of net revenues do I get?

  • You get 70% of the net revenues (excl. VAT) or a lower percentage in case the Minimum Commission is applicable in the release territory
  • There is no performance threshold for games sold, meaning you get paid for every unit sold.

Can I update my game with patching and downloadable content?

  • Nintendo has a flexible patching policy.
  • Independent developers have the same capabilities as publishers in regard to eCommerce functionality on Nintendo platforms.
  • Please see the documentation on your desired platform Guidelines for more details.
